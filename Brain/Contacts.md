# How to use this file

- **Drafting an email or Slack message to anyone listed:** check this file first for the canonical address and Slack ID. Do not guess. If a person is missing, that is a flag, not a licence to make up an address.
- **Maddie says a colloquial name** ("can you message Lynda about X"): the Aliases column maps colloquial to canonical.
- **Staff vs Clients & External** classification matters because it changes how skills behave. Staff get internal-tone drafts, Clients & External get more formal drafts and trigger compliance checks (signed release, disclosure label, etc.). Skills like [[Talent/Ambassador Brief]] read this file to know which path to take.
- **New person joins Fade Group as staff:** add their row in the Staff section immediately, even if fields are TBC.
- **New ambassador, contractor, or external partner:** add to Clients & External section with status (in negotiation / signed / active / wound down).
- **Someone leaves Fade Group:** update their row with departure date and status, do not delete. Pattern matches Claire Green's row above.
- **Status changes** (e.g. Luke's ambassador deal signs, Katie's working relationship confirms): update the Status line and move their row between sections if needed.
